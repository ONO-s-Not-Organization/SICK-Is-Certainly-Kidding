# 表格合并逻辑重构方案

本方案将原本的区域合并功能细化为“向右合并”和“向下合并”，并优化底层渲染以支持更复杂的跨行布局。

## 1. 合并功能细化
- **向右合并 (Merge Right)**：
  - 将当前选中单元格与右侧相邻单元格合并。
  - 逻辑：增加当前单元格的 `colSpan`，并标记右侧单元格为 `hidden`。
- **向下合并 (Merge Down)**：
  - 将当前选中单元格与下方相邻单元格合并。
  - 逻辑：增加当前单元格的 `rowSpan`，并标记下方单元格为 `hidden`。
- **取消合并**：增加一个按钮用于恢复单元格的原始状态。

## 2. 渲染架构升级 (CSS Grid)
- 为了完美支持 `rowSpan`（向下合并），我们将把渲染引擎从 Flexbox 升级为 **CSS Grid**。
- **Grid 布局优势**：
  - 允许单元格通过 `grid-row: span X` 真正跨越物理行。
  - 保持外层容器的对齐一致性。
  - 依然保持之前实现的“填充空隙”和“医疗表单”风格。

## 3. 交互更新
- 在单元格选中的浮动工具栏中，添加三个新按钮：
  - `MergeRight` (图标：向右箭头 + 合并)
  - `MergeDown` (图标：向下箭头 + 合并)
  - `Split` (取消合并)
