# 诊断证明生成工具开发方案

## 1. 技术栈选型
- **前端框架**: React (Vite 驱动，保证极速开发体验)
- **样式处理**: Tailwind CSS (快速构建高度自定义的 UI)
- **状态管理**: Zustand (轻量级，适合处理复杂的编辑器状态)
- **表格核心**: `react-spreadsheet` 或自定义可编辑表格组件 (针对诊断证明的特定布局优化)
- **截图导出**: `html2canvas` (将 DOM 直接渲染为图片)
- **图片处理**: `react-rnd` (用于实现图片的拖拽和缩放)
- **本地存储**: `idb-keyval` (基于 IndexedDB，存储容量比 localStorage 更大，适合存图片数据)

## 2. 核心模块设计

### A. 编辑器画布 (Canvas Area)
- 模拟真实的 A4 纸张比例。
- 支持网格参考线，方便对齐。

### B. 类 Excel 表格组件
- **单元格编辑**: 双击进入编辑模式。
- **动态行列**: 提供快捷键或右键菜单增加/删除行列。
- **样式控制**: 支持设置单元格边框、背景色、对齐方式。

### C. 资源管理
- 支持图片拖入、粘贴上传。
- 文本块支持富文本简易编辑。

### D. 导出引擎
- 优化高 DPI 导出，确保生成的图片清晰。

## 3. 目录结构建议
```text
src/
├── components/       # 通用 UI 组件
│   ├── Editor/       # 编辑器核心组件
│   │   ├── Canvas.tsx
│   │   ├── Toolbar.tsx
│   │   ├── Table.tsx
│   │   └── ImageBox.tsx
├── store/            # 状态管理 (Zustand)
├── hooks/            # 自定义 Hook (如导出逻辑、快捷键)
├── utils/            # 工具函数 (html2canvas 封装)
└── App.tsx
```

## 4. 进度安排
1. **第一阶段**: 项目初始化，搭建基础布局和 Canvas 容器。
2. **第二阶段**: 开发文本和图片拖拽组件。
3. **第三阶段**: 攻克表格编辑组件（核心难点）。
4. **第四阶段**: 实现本地存储备份和图片导出。
5. **第五阶段**: 细节打磨（撤销/重做、模板预设）。
